Version 0.9.1 Released

  • No need to wire up the Bootstrapper anymore, now Twine reads twine.properties inside component jars
  • Fixed a ClassCastException when specifying an int return type in Idoc functions
  • Cleaner handling of Float, null and void return types from Idoc functions

Why Twine?

  • Write UCM services, Idoc script extensions and filters using plain Java objects and annotations
  • Dependency injection reduces coupling, promotes cleaner methods and allows unit testing
  • Stop writing that same old boilerplate validation code. Ask for a Date or ResultSet and Twine will parse and validate it for you

Examples

public class ExampleScriptPackage {

  @IdocFunction
  public String greet(String message, UserData user) {
    return message + " " + user.getProperty("dFullName") + "!";
  }
}
// usage: <!--$ greet("Hello") -->
// result: Hello Tim Stirrat!

Stop writing that same old parameter checking code:

public class ExampleServicePackage {

  @ServiceMethod(name = "EXAMPLE_SERVICE")
  public void example(@Binder(name = "date") Date date) {
    // do something with date
  }
}
// usage: ?IdcService=EXAMPLE_SERVICE&date=Hello
// result: Error: Parameter 'date' must be parseable as a date
